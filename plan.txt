Build an android app to read out text. User types or pastes text into a text box, then play the text as audio.
This app will run on my phone (redmi k90 pro max) with qualcomm snapdragon CPU.
Use android integrated TTS service with plugable engines (google speech synthesis, kokoro tts, etc).
I will use kokoro for more natural voice, but also want to keep power consumption as low as possible. Hope quantized model (e.g., int8) can meet my expectation (details below).
Only english voice is necessary.


UI requirement
--------------
- a text box user can type or paste text in it, no size limitation
- a play/pause button to start/pause audio playback
- stop button to stop the playback
- starting and stoping may experience some delay, show info in UI to inform user
- continue playing when screen is shut off
- a dropdown list to select speakers provided by related kokoro model, through standard android tts service
- a checkbox to enable slower playback, at 0.75 normal speed
- remember settings and restore on next startup
- input text box is disabled during playback, and it will highlight the current playing text block in middle of visible area, remember the text and current position, restore on startup
- focus to text box at stopped state, use can scroll text, move cursor, then the playback shall start at that position
- audio playback must be smooth. It's okay for some latency at the beginning, but there should be no delay once it's playing. Figure out approaches to meet the requirement (e.g., cut text to small chunks at sentence boundary to interleave conversion and playback). kokoro computation is non-trivial. Per my test, 1 producer thread with queue size = 4, run at background priority on performance core, is good enough on my phone.
- handle carefully possible underrun and other edge cases, avoid pop noises
- remove temporary files (if any) on close and at startup (in case app crashed), also during the playback if a temp file won't be used later


kokoro tts
----------
I plan to use int8 quantized model (model_quantized.onnx) from https://huggingface.co/onnx-community/Kokoro-82M-v1.0-ONNX. But you are free to suggest better approach, the only concern is lower power consumption.
I tried all the quantized model voice sample, all are great. I didn't find android example how to use it.

https://github.com/DevGitPit/Kokoros releases apk which integrate kokoro into android tts service, convenient for my case. But the problem is it uses the fp32 model, which consumes too much power per my test. I want to use quantized version like int8.
It will be great if we can modify that apk to meet my requirement. And I'm not sure if android tts service can support quantized model, or my app shall handle it.

I also found an android app uses int8 kokoro (https://github.com/puff-dayo/Kokoro-82M-Android), but not sure if it's based on android tts service or runs on kokoro directly, and whether it handles phoneme (it looks just a simple demo).

Note that if you will build onnxruntime, make sure to use the 1.20.0 version. Newer version uses Arm SME instruction which is not supported by my phone.
