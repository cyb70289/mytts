# MyTTS - Build & Test Guide

## Prerequisites

- **macOS** (tested on Apple Silicon M4)
- **Android Studio** (Ladybug or newer) with:
  - Android SDK 35
  - Build-tools 34.0.0 (auto-installed by Gradle)
  - NDK (auto-installed by Gradle on first build -- typically 27.x or 28.x)
  - CMake 3.22.1 (install via SDK Manager > SDK Tools > CMake)
- **Java 21** (bundled with Android Studio)

### Environment Variables

Add to your shell profile (`~/.zshrc` or `~/.bashrc`):

```bash
export ANDROID_HOME="$HOME/Library/Android/sdk"
export JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"
export PATH="$ANDROID_HOME/platform-tools:$ANDROID_HOME/emulator:$PATH"
```

Reload: `source ~/.zshrc`

## Download Model Assets

The ONNX model, voice files, and eSpeak-ng data are too large for git. Download them before building:

### 1. Kokoro int8 quantized model (~92 MB)

```bash
mkdir -p app/src/main/assets
curl -L "https://huggingface.co/onnx-community/Kokoro-82M-v1.0-ONNX/resolve/main/onnx/model_quantized.onnx" \
  -o app/src/main/assets/model_quantized.onnx
```

### 2. Voice style files (.bin)

Kokoro v1.0 ONNX voices are raw float32 `.bin` files (522 KB each, no header).

```bash
mkdir -p app/src/main/assets/voices

# Download all English voices from Kokoro v1.0
for voice in af_heart af_alloy af_aoede af_bella af_jessica af_kore af_nicole af_nova af_river af_sarah af_sky am_adam am_echo am_eric am_liam am_michael am_onyx am_puck am_santa bf_alice bf_emma bf_lily bm_daniel bm_fable bm_george bm_lewis; do
  echo "Downloading ${voice}.bin ..."
  curl -sL "https://huggingface.co/onnx-community/Kokoro-82M-v1.0-ONNX/resolve/main/voices/${voice}.bin" \
    -o "app/src/main/assets/voices/${voice}.bin"
done
```

Or download only a few voices to reduce APK size (each is ~512 KB):

```bash
# Minimal set - one female, one male
for voice in af_heart am_adam; do
  curl -sL "https://huggingface.co/onnx-community/Kokoro-82M-v1.0-ONNX/resolve/main/voices/${voice}.bin" \
    -o "app/src/main/assets/voices/${voice}.bin"
done
```

### 3. eSpeak-ng data files (English only, ~800 KB)

eSpeak-ng requires pre-compiled binary data files for phoneme lookup. These must be
generated by building eSpeak-ng natively, then copying the output.

```bash
# Install cmake if not in PATH (or use the Android SDK's cmake)
CMAKE="$ANDROID_HOME/cmake/3.22.1/bin/cmake"

# Download the eSpeak-ng fork (same version used by the app's NDK build)
curl -L -o /tmp/espeak-ng-src.zip \
  "https://github.com/csukuangfj/espeak-ng/archive/f6fed6c58b5e0998b8e68c6610125e2d07d595a7.zip"
unzip -q -o /tmp/espeak-ng-src.zip -d /tmp/espeak-ng-src

ESRC=/tmp/espeak-ng-src/espeak-ng-f6fed6c58b5e0998b8e68c6610125e2d07d595a7

# Build natively to generate the data files
mkdir -p /tmp/espeak-ng-build
"$CMAKE" -S "$ESRC" -B /tmp/espeak-ng-build \
  -DBUILD_ESPEAK_NG_EXE=ON \
  -DBUILD_ESPEAK_NG_TESTS=OFF \
  -DUSE_ASYNC=OFF -DUSE_MBROLA=OFF -DUSE_LIBSONIC=OFF \
  -DUSE_LIBPCAUDIO=OFF -DUSE_KLATT=OFF -DUSE_SPEECHPLAYER=OFF
"$CMAKE" --build /tmp/espeak-ng-build -j$(sysctl -n hw.ncpu)

# Copy only what's needed (English only)
DEST=app/src/main/assets/espeak-ng-data
mkdir -p "$DEST/lang/gmw" "$DEST/voices/!v"
cp /tmp/espeak-ng-build/espeak-ng-data/{phondata,phonindex,phontab,intonations,en_dict} "$DEST/"
cp "$ESRC/espeak-ng-data/lang/gmw/en" "$ESRC/espeak-ng-data/lang/gmw/en-US" "$DEST/lang/gmw/"
```

### Verify assets

```bash
ls -lh app/src/main/assets/
# Should show:
#   model_quantized.onnx    (~92 MB)
#   espeak-ng-data/         (directory, ~800 KB total)
#   voices/                 (directory)

ls app/src/main/assets/voices/
# Should show .bin files (522240 bytes each)

ls app/src/main/assets/espeak-ng-data/
# Should show: en_dict  intonations  lang  phondata  phonindex  phontab  voices
```

## Build

### Debug APK

```bash
./gradlew assembleDebug
```

Output: `app/build/outputs/apk/debug/app-debug.apk`

The first build will:
1. Download the eSpeak-ng source via CMake FetchContent (~17 MB)
2. Cross-compile eSpeak-ng as a static library for arm64-v8a
3. Build the JNI bridge (`libespeak_jni.so`)
4. Compile the Kotlin code and package the APK

Subsequent builds are incremental and much faster (~2-5 seconds).

> **Tip:** To speed up the first build, pre-download the eSpeak-ng archive:
> ```bash
> cp /tmp/espeak-ng-src.zip /tmp/espeak-ng-f6fed6c58b5e0998b8e68c6610125e2d07d595a7.zip
> ```
> CMake will find the cached file and skip downloading.

### Signed Release APK

The project includes a `release.keystore` for convenience (password: `mytts123`).
For production distribution, generate your own keystore.

```bash
./gradlew assembleRelease
```

Output: `app/build/outputs/apk/release/app-release.apk`

### Verify release signing

```bash
$ANDROID_HOME/build-tools/34.0.0/apksigner verify --verbose --print-certs \
  app/build/outputs/apk/release/app-release.apk
```

Should show `Verified using v2 scheme (APK Signature Scheme v2): true`.

### Clean build

```bash
./gradlew clean assembleRelease
```

## Native Code (eSpeak-ng)

The app uses [eSpeak-ng](https://github.com/espeak-ng/espeak-ng) for grapheme-to-phoneme
conversion, compiled as a static NDK library and called via JNI. Only the text-to-phoneme
functionality is used -- no audio synthesis from eSpeak.

### How it works

1. **Build time**: CMake fetches the [csukuangfj fork](https://github.com/csukuangfj/espeak-ng)
   (commit `f6fed6c`, same as sherpa-onnx) and cross-compiles it for arm64-v8a as a static
   library. Audio/synth features are all disabled. The JNI bridge (`espeak_jni.cpp`) is
   compiled into `libespeak_jni.so` and bundled in the APK.

2. **First launch**: `EspeakBridge` extracts `espeak-ng-data/` from APK assets to the
   app's internal storage, then calls `espeak_Initialize()` pointing to that directory.
   Subsequent launches skip extraction (version marker file).

3. **Phonemization**: `PhonemeConverter` calls `EspeakBridge.textToPhonemes(text)` which
   invokes `espeak_TextToPhonemes()` in IPA mode with `^` tie characters. The raw IPA
   output is then minimally normalized (e.g. `r` → `ɹ`, `ɚ` → `əɹ`) to match
   the Kokoro model's expected phoneme set.

### Key constraints

- **ONNX Runtime 1.20.0** is pinned in `app/build.gradle.kts`. Newer versions use ARM SME
  instructions that crash on the target device.
- **eSpeak-ng is only for phonemization** -- the ONNX model generates all audio.
- **English only** -- only `en_dict` and `lang/gmw/en*` data files are bundled.

## Generating Your Own Signing Key

For production releases, replace the included keystore:

```bash
keytool -genkeypair -v \
  -keystore my-release-key.keystore \
  -alias my-key-alias \
  -keyalg RSA -keysize 2048 \
  -validity 10000

# Then update app/build.gradle.kts signingConfigs.release with your values
```
